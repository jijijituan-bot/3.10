# 🚀 直接使用指南

## ⚠️ Railpack 环境问题

由于 Railpack 环境的限制，服务器启动可能遇到问题。

## ✅ 简单解决方案

### 方法1：直接打开 HTML 文件（推荐）

**无需服务器，直接使用：**

1. **图片上传工具**
   ```
   双击打开 index.html
   ```
   - ✅ 上传图片到 ImgBB
   - ✅ 生成永久 URL
   - ✅ 科技感界面

2. **视频转GIF工具**
   ```
   双击打开 video-to-gif-offline.html
   ```
   - ✅ 4种解决方案
   - ✅ 在线工具推荐
   - ✅ 完整使用指南

3. **配置助手**
   ```
   双击打开 config.html
   ```
   - ✅ API Key 配置
   - ✅ 代码生成器

### 方法2：使用其他端口

如果需要服务器，修改 app.js 中的端口：

```javascript
const PORT = process.env.PORT || 3000; // 改为 3000
```

然后运行：
```bash
node app.js
```

### 方法3：使用 Python 服务器

```bash
python -m http.server 3000
```

然后访问：`http://localhost:3000`

## 🎯 推荐使用流程

### 图片上传：
1. 双击 `index.html`
2. 上传图片
3. 复制生成的永久 URL

### 视频转GIF：
1. 双击 `video-to-gif-offline.html`
2. 选择方案1（在线工具）
3. 点击 EZGIF 或其他工具链接
4. 在线转换视频为GIF
5. 下载GIF后，回到图片上传工具上传获取永久链接

## 💡 为什么推荐直接打开 HTML？

- ✅ **无需配置** - 直接可用
- ✅ **无端口冲突** - 不依赖服务器
- ✅ **功能完整** - 所有功能都能正常使用
- ✅ **兼容性好** - 任何浏览器都支持

## 🔧 如果遇到跨域问题

某些功能（如 ImgBB 上传）可能需要 HTTPS 或服务器环境。

**解决方案：**
1. 使用现代浏览器（Chrome、Edge、Firefox）
2. 或者使用在线版本的工具

## 📱 移动端使用

所有页面都支持移动端，可以在手机浏览器中直接打开使用。

---

**总结：直接双击 HTML 文件即可使用，无需复杂配置！** 🎉
